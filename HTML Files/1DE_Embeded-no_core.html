1D Gel Electrophoresis



INSTRUCTIONS:


  1. Mark the checkboxes for all standards you would like to include.
  2. Click a well for your protein and navigate to your protein's data file (FASTA) in the file explorer window. You can download the files for the DNASU BASIL starter pack V1 here.
  3. Set your desired voltage and % Acrylamide.
  4. Run the simulation by clicking start and stop it right before the dye runs to the bottom of the gel.
  5. Analyze the run by clicking on any band to display its contents data and exploring the plot generated in the second simulation tab.
  6. Either refill wells to run it again with the same proteins or clear wells to restart the simulation.


You can open the module instructions in a separate tab here or use the embedded page below



The default version of this 1D electrophoresis simulation is based on ten different protein sets that are part of the BASIL (Biochemistry Authentic Scientific Inquiry Lab). If you are interested, you can also design your own separations. Instructors can use this to prepare their students for a wet lab experience, such as protein purification. Students can use this to enhance their understanding or to dig deeper as they analyze data for a lab report.


Instructions for Creating Your Own Separation Lab


  1. The standards are hard-coded into the simulation. They cover a MW range that will apply to most separations, especially at the undergraduate level.

  2. All protein sequences must be in FASTA format and will look something like this in a text editor.

    >2O14_1|Chain A|Hypothetical protein yxiM|Bacillus subtilis (1423) MFGGIENVKAAEPKVYQFDFGSGSMEPGYIGVRASDRYDRSKGY GFQTPENMRDVAASGAGVKSDAVEFLAYGTKSNNTFNVDLPNGL YEVKVTLGNTARASVAAEGVFQVINMTGDGAEDTFQIPVTDGQL NLLVTEGKAGTAFTLSALKIKKLSDQPVTNRTIYVGGDSTVCNYY PLNSSKQAGWGQMLPHYIDKHTFQVRNMASGGQIARGFRNDGQ LEAILKYIKPGDYFMLQLGINDTNPKHKESEAEFKEVMRDMIRQ VKAKGADVILSTPQGRATDFTSEGIHSSVNRWYRASILALAEEE KTYLIDLNVLSSAYFTSIGPERTLGLYMDGDTLHPNRAGADALA RLAVQELKRQGIAGFLEHHHHHH

  3. In the 1DE simulation, all of the bands have a default depth of 1 pixel. To make a band broader, you simply add “ |$#” followed by a number ranging from 2 - 9. So if the protein listed above is the target protein for your separation, you can change the FASTA file to look like this:

    >2O14_1|Chain A|Hypothetical protein yxiM|Bacillus subtilis (1423) |$#4 MFGGIENVKAAEPKVYQFDFGSGSMEPGYIGVRASDRYDRSKGY GFQTPENMRDVAASGAGVKSDAVEFLAYGTKSNNTFNVDLPNGL YEVKVTLGNTARASVAAEGVFQVINMTGDGAEDTFQIPVTDGQL NLLVTEGKAGTAFTLSALKIKKLSDQPVTNRTIYVGGDSTVCNYY PLNSSKQAGWGQMLPHYIDKHTFQVRNMASGGQIARGFRNDGQ LEAILKYIKPGDYFMLQLGINDTNPKHKESEAEFKEVMRDMIRQ VKAKGADVILSTPQGRATDFTSEGIHSSVNRWYRASILALAEEE KTYLIDLNVLSSAYFTSIGPERTLGLYMDGDTLHPNRAGADALA RLAVQELKRQGIAGFLEHHHHHH

  4. Create the proteins samples (in FASTA format) for the wells for a protein purification. You can put multiple FASTA sequences in a single file. Just make sure that you start each protein with the line that has a ‘>’ sign on the beginning.

    • Well 1: standards (hard-coded default)
    • Well 2: the full proteome of the organism, for example, E. coli
    • Well 3: the full proteome of the organism with the overexpressed protein band labeled with the tag “|$#6” on the first line of the FASTA file to make that band thicker.
    • Well 4: the result of the first separation step. If this is cation exchange chromatography at pH 4.8, then you’ll need to determine which proteins are positively charged at pH 4.8. You can use this tool (https://www.protpi.ch/Calculator/ProteinTool/) but it only does one protein at a time. You may be able to find other tools.
    • Well 5: the result of the next separation step. For example, you could do gel filtration select proteins based on MW (the number of amino acids in the protein chain is a close approximation that you could use for this separation).
    • Well 6: first wash of a metal ion affinity chromatography step - should release all proteins except the target protein.
    • Well 7: second wash of a metal ion affinity chromatography step.
    • Well 8: elution wash #1 with imidazole of the metal ion affinity chromatography step. Set the band width at |$#5.
    • Well 9: elution wash #2 with imidazole of the metal ion affinity chromatography step. Set the band width at |$#8.
    • Well 10: elution wash #3 with imidazole of the metal ion affinity chromatography step. Set the band width at |$#3.

  5. Put all of the proteins in a single file that you share with your students.

  6. Have them use the 1DE simulation with the protein samples in your folder.

Other possibilities would be any mixture of proteins you normally separate on an SDS-PAGE gel in your lab. For example, you could create protein samples that are generated by different experiments that may or may not express a target protein. Or you could use it to teach about the molecular weights of proteins that are monomeric and proteins that are multimeric. You would just need to create set up two versions of the multimeric protein - one in which each separate chain had its own FASTA file and one in which all of the chains are in a single FASTA file.


Related Content


  • NCBI Index of Archaea Genomes

    NCBI Index of Bacteria Genomes

    These resources catalog multiple reference sequences of known archaea and bacteria genomes, respectively. The protein files contained in this database can be downloaded and run in the 1D Electrophoresis Simulation. Follow the steps below:

    1. Click on your desired archaea or bacteria genome in the sequence index>
    2. Select “latest_assembly_versions/”
    3. Select the newest version of your desired sequence by comparing the “Last modified” results
    4. From the resulting list, choose the result ending in “protein.faa.gz” This will automatically download the file to your computer.
      • For Windows and Linux users: You will need to convert the file from a .gunzip (gz) to .faa. If you do not already have a file converter, 7-zip is a free software that can unzip .gunzip files. You may use other file conversion programs as well, like WinZip.
        • Download 7-zip here.
        • Open 7-zip and navigate to your .gunzip file. Select the file and click “Extract” in the top ribbon
        • Save the extracted .faa file somewhere secure
      • For Mac Users: Mac is able to extract .gz files without external aid. Ensure the file has been unzipped to .faa format before proceeding to the simulation
    5. The .faa file can now be loaded into the simulation. Load selected standards into well 1 before loading your .faa file(s) into the remaining wells.
    6. Run the simulation.

  • HAMAP Proteomes

    HAMAP is a protein annotation system that compiles protein family profiles from UniProtKB. You can use HAMAP to acquire proteome information from a variety of archaea and bacteria. Follow the steps below:

    1. Navigate to the HAMAP website. Depending on your desired proteome target, choose either the archaea or bacteria drop down menu
    2. Find and select your desired organism either by name or UniProt. Once selected, choose “Proteome Info” on the right-hand side of the page. This will open up a page in UniProt. This page provides some basic information about your selected target.
    3. Scroll down to the section titled “Components.” Select the sequence checkboxes and hit “Download”
    4. On the next screen, make sure the format is listed as “FASTA (canonical)”
    5. From here, there is an option to compress the file. Compressing the file leads to a faster download, but you will have to unzip the file after it is downloaded. For instructions on how to do this, see step 4 of how to obtain files from NCBI Index sources above. If you do not wish to extract the files, download them uncompressed.
    6. The .fasta file can now be loaded into the simulation. Load selected standards into well 1 before loading your .fasta file(s) into the remaining wells.
    7. Run the simulation.

    In the Spring of 2022, RIT student Tyler Bernhardt adapted a previous SDS-PAGE simulation by expanding well capacity, updating sample loading UI, integrating ten built-in protein sets and learning objectives designed for the BASIL curriculum, and implementing user ability to load custom designed protein mixtures.
